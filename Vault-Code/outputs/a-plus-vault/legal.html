Privacy Notice

A+ Vault is private by default. We collect only what is needed for the user-selected action: account details, saved Vault objects, source/context metadata, notes, collections, projects, moodboards, basic security logs, and settings.

Data we process

  • Account data: email, login method, profile settings, storage usage.
  • User content: images, screenshots, links, notes, videos, titles, keywords, and moodboard objects.
  • Source context: source URL, domain, page title, capture method, saved date, preview URL where available.
  • Extension data: only the item the user chooses to save, plus local recent captures and settings.

Purpose and user control

We use data to store private references, search, organize, preview, sync, secure the account, and provide user-requested sharing. Users can delete objects, clear local data, export alpha data where available, and request access or deletion at privacy@aplus1.app.

We do not claim ownership of user content. Saving a reference does not grant copyright, trademark, publicity, or usage rights.

Alpha Terms

A+ Vault is an alpha product for creative reference management. Features may change, fail, or lose prototype-only local data. The service may not capture every website, file, or preview perfectly.

User content license

Users retain ownership of their user content. A+ Vault receives a limited, non-exclusive license only to operate the service: store, back up, resize, create thumbnails, process for search, display to the user, and display to people the user explicitly shares with.

User responsibility

Users are responsible for having the rights needed to upload, save, share, publish, or reuse references. Private reference storage is not a license to copy, publish, sell, or commercially exploit third-party work.

Copyright & Takedown Policy

A+ Vault preserves source and context to support responsible reference use. Public sharing is explicit and should include source attribution where available. Source credit alone does not make copying or public display lawful.

Report copyright infringement

Send reports to copyright@aplus1.app with: your name/contact, the original work, the A+ Vault URL/object, a description of the issue, a good-faith statement, an accuracy statement, and your electronic signature.

Process

  • Review complaint completeness and log timestamps.
  • Disable public access where appropriate.
  • Notify the account owner when required.
  • Allow counter-notice where applicable.
  • Apply repeat-infringer restrictions to sharing or accounts.

Acceptable Use Policy

Do not use A+ Vault to infringe copyright, bypass paywalls or DRM, collect private data without rights, publish confidential client materials, upload malware, harass, stalk, impersonate, scrape at scale, or interfere with systems.

The browser extension must be used only through explicit user action. It must not track browsing history, read unrelated form fields, or collect unrelated page content in the background.

AI Processing Notice

AI Lite in the alpha may generate keywords, colors, OCR placeholders, and summaries to make references searchable. Production AI processing must disclose provider, data sent, retention, training policy, and user controls before third-party AI processing is enabled.

  • Default: private objects stay private.
  • User control needed: disable AI per item/project before external AI is used.
  • No training promise should be made until vendor contracts confirm it.

Export, Deletion & Privacy Requests

Users should be able to access, correct, export, and delete their data. In the alpha, deletion removes local objects from the workspace. Production deletion must also remove database rows, storage files, thumbnails, share links, and queued AI processing.

Target retention schedule

  • Vault objects: until user deletes or closes account.
  • Trash: 30 days when trash is implemented.
  • Backups: disclose actual backup deletion window before production.
  • Security logs: short, purpose-limited retention.

Privacy requests: privacy@aplus1.app

Chrome Extension Privacy

The A+ Vault Capture extension saves creative references only when you explicitly choose an action: right-click + Keep in Vault, + Snapshot to Vault, or the popup capture buttons.

Data sent per capture

  • Page URL and title
  • Selected image, video, or link URL
  • Optional title, note, and collection choice
  • Cropped snapshot image for snapshot capture

Where data goes

  • Production demo: https://aplus-vault.vercel.app/api/vault/*
  • Local development: http://127.0.0.1:5177/api/vault/*

Your extension sync token and recent capture previews stay in Chrome local storage on your device. The extension never stores database secrets or service-role keys.

What we do not do

  • No background browsing history collection
  • No hidden page scraping
  • No automatic publishing of saved references

Contact: privacy@aplusvault.app
