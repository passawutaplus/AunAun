# Aplus1 MVP Strategy

Accessed: 2026-07-03

## MVP Thesis

Aplus1's MVP should prove one loop:

> A creator publishes a real project with useful context, a hirer discovers it, and an opportunity conversation starts from that project.

If this loop works, Aplus1 has a real product. If it does not, adding jobs, payments, contests, AI ranking, or admin complexity will only hide the core uncertainty.

## First User Segment

### Primary Creator Segment

Emerging-to-working Thai creatives with at least 3 portfolio-worthy works.

Examples:

- graphic designers
- illustrators
- UI/UX designers
- motion/video editors
- photographers
- branding/social content designers
- 3D/visual artists

Why this segment:

- They already have visual output.
- They feel portfolio pain.
- They are likely to share links.
- They can benefit from paid and non-paid opportunities.
- They are underserved by resume-first job boards.

### Primary Opportunity Giver Segment

SMEs, founders, marketing teams, and small agencies looking for creative fit.

Why this segment:

- They often cannot evaluate creative talent from resume alone.
- They care about output and taste fit.
- They are frustrated by mismatch between sales talk and actual delivery.
- They need a lighter path than agency procurement or generic job posts.

## North Star Metric

Recommended:

> Project-qualified opportunity conversations per week.

Definition:

- A conversation/inquiry that is started from a specific project page or project card.
- Must include opportunity type or brief context.
- Excludes generic spam/contact.

Why:

- It connects work to opportunity.
- It avoids vanity metrics like uploads or views only.
- It can work for paid and non-paid opportunities.

## Activation Metrics

Creator activation:

- publishes 1 project with cover, category, role, and at least 2 context fields
- sets opportunity status
- receives first meaningful view/save/follow/contact

Hirer activation:

- views at least 3 project pages
- saves at least 1 project/creator or sends 1 project-based inquiry

## Retention Signals

Creator:

- publishes second project within 14 days
- updates opportunity status
- returns after notification
- shares project link externally

Hirer:

- returns to saved list
- views more projects in same style/category
- sends another inquiry
- uses Aplus1 for a second brief

## Feature Priority

### Must-Have Before Launch

| Feature | Why | Pain Solved | Metric |
|---|---|---|---|
| Creator profile | identity hub | scattered online identity | profile completion |
| Project upload | core proof object | no structured place for proof | first project published |
| Project context template | builds trust | image-only portfolios lack explanation | context field completion |
| Opportunity status | broadens beyond "รับงาน" | creators have different needs | status set rate |
| Contact from project | converts work to opportunity | generic contact loses context | project-to-contact rate |
| Save project/creator | supports non-immediate hiring | hirers compare and return | save rate |
| Search/discovery by category/style | helps hirers find fit | feed alone is noisy | search-to-project view |
| Basic moderation/report | trust and safety | public platform risk | reports resolved |
| Shareable project page | distribution | creators already use social | external referral traffic |
| Not-found/error states | resilience | blank pages destroy trust | error recovery rate |

### Should-Have Soon After Launch

| Feature | Why |
|---|---|
| AI case-study helper | reduces writing friction, but must preserve creator voice |
| private/unlisted project | helps creators share sensitive work |
| simple testimonial | increases trust after opportunities happen |
| creator onboarding checklist | helps first upload |
| hirer onboarding | teaches how to search from work |
| related/similar work | improves discovery |
| availability filters | helps hirers act faster |
| opportunity inbox labels | separates paid/collab/internship |

### Nice-To-Have

| Feature | Why Later |
|---|---|
| full payment escrow | heavy ops/legal; prove demand first |
| contest system | risks spec work; needs trust policy |
| mentor marketplace | valuable but separate liquidity loop |
| public follower social network | can distract from opportunity loop |
| advanced ranking AI | dangerous before enough quality data |
| agency roster product | good B2B later after creator supply |
| portfolio import automation | useful for scale but not initial proof |

### Not Now

- broad job board competing with JobsDB
- full marketplace package pricing competing with Fastwork
- crypto/NFT-style portfolio features
- complex gamification
- public popularity ranking that makes juniors feel invisible

## Launch Strategy

### Wedge Category

Start with 2-3 verticals:

1. Graphic/branding/social media design.
2. Illustration/visual art.
3. UI/UX/web design.

Why:

- Visual proof is easy to judge.
- Hirers have frequent needs.
- Creators already share work socially.
- Categories overlap with Fastwork/Behance/Dribbble but Aplus1 can localize.

### Supply Seeding

Goal before public push:

- 100 quality creators
- 300 quality projects
- at least 30 projects with strong context
- 20 creators open to paid work
- 20 creators open to collaboration
- 20 students/juniors open to internship/feedback

Quality bar:

- cover image
- role
- category
- opportunity status
- short context
- no empty/generic projects

### Demand Seeding

Recruit:

- 20 SME/founder/marketer viewers
- 10 agency/studio people
- 10 creator leads who can invite collaborators

Ask them to complete tasks:

- find 3 creators for a sample brief
- save 3 projects
- send 1 inquiry from a project
- rate trust level

## Launch Campaign

Theme:

> ผลงานพาไปเจอโอกาส

Campaign mechanics:

1. "เปิดรับโอกาส" creator week.
2. Feature creators by opportunity type, not just category.
3. "From this work to next opportunity" stories.
4. Invite agencies/SMEs to browse curated collections.
5. Social post templates for creators:
   - "ตอนนี้เปิดรับโอกาสด้าน..."
   - "ดูผลงานและคุยโอกาสจากชิ้นนี้ได้ที่..."

## Measurement Dashboard

| Funnel | Metric |
|---|---|
| Acquisition | creator signup, hirer visitor, external share visits |
| Activation | first project published, opportunity status set, first 3 project views |
| Discovery | project impressions, project detail views, search/filter use |
| Trust | context field completion, profile-to-project depth, save rate |
| Opportunity | project-based inquiries, reply rate, accepted opportunities |
| Retention | second project, saved-list return, repeat inquiry |

## Main Risks

| Risk | Why It Matters | Mitigation |
|---|---|---|
| Too broad | no liquidity in any vertical | seed 2-3 verticals first |
| Looks like generic portfolio | no reason to switch | emphasize opportunity status + contact from project |
| Looks like marketplace | dragged into price competition | avoid package-first UI |
| Low-quality uploads | harms trust | onboarding, examples, moderation, project template |
| Creators fear copying | reduces uploads | watermark/private/unlisted later; clear rights copy now |
| Hirers do not contact | discovery without conversion | project-specific CTA and guided inquiry |
| Juniors feel intimidated | loses early community | student/experimental mode |

## 30-Day Validation Plan

Week 1:

- recruit 20 creators
- run 8 interviews
- test copy/CTA
- publish 30 seeded projects

Week 2:

- recruit 10 hirers/agencies
- run project discovery tasks
- measure saves/contact intent
- fix project page context

Week 3:

- launch closed beta
- push creator social sharing
- collect first inquiries
- classify opportunity types

Week 4:

- review metrics
- decide strongest vertical
- cut features that do not drive project-based opportunities
- prepare public beta message

## Decision Rules

Proceed if:

- 50%+ onboarded creators publish at least 1 project.
- 30%+ creators set opportunity status.
- 20%+ hirers save at least 1 project/creator.
- At least 10 project-based inquiries happen in beta.
- Users can explain difference from Fastwork/Behance in their own words.

Pivot positioning if:

- creators say "โอกาส" is too vague after explanation
- hirers only want job-board behavior
- project context does not improve trust

Pause feature expansion if:

- upload completion is below 30%
- project-to-contact is near zero
- most inquiries are spam or very low quality

## Final MVP Recommendation

Build the smallest strong version of:

**project portfolio + opportunity status + project-specific contact + save/discovery**

Do not build a full marketplace yet. The strategic proof is whether Aplus1 can make a real project create a real opportunity conversation.
