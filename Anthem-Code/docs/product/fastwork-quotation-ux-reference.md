## 1) สร้างเทมเพลต (Create Template)

Modal ขาว กะทัดรัด ฟิลด์เรียงแนวตั้ง:

| ฟิลด์ | โน้ต UX |
|------|---------|
| ชื่องาน | จำง่าย — placeholder สั้น |
| คำอธิบายงาน | textarea + ตัวนับ ~2,000 |
| สิ่งที่ลูกค้าจะได้รับ | deliverables / ไฟล์ / จำนวน |
| การแก้ไขงาน | ตัวเลข + suffix «ครั้ง» ในช่อง |
| หมายเหตุ | เงื่อนไขเพิ่มเติม (ฝั่งลูกค้า) |

Footer: **ยกเลิก** (เทา) · **สร้างเทมเพลต** (ดำ/หลัก)

**ของเรา (ตัดสินใจแล้วในร่าง):** แยก 2 หมายเหตุ  
- ฝั่งลูกค้า → โชว์บนใบเสนอราคา  
- ฝั่งเรา (ภายใน) → เห็นเฉพาะผู้ส่ง / หลังบ้าน ไม่โชว์บนเอกสาร

## 2) เทมเพลต / ใบแบบ «รอบเดียว»

- แบนเนอร์อ่อน + ไอคอนเอกสาร/เงิน: **ใบเสนอราคาแบบ 'รอบเดียว'**
- ช่องชื่อสั้นใต้แบนเนอร์ (เช่น งานออกแบบโลโก้)
- **ระบุราคา** + suffix «บาท»
- บล็อกพับได้ **ข้อมูลเพิ่มเติม**: ชื่องาน, คำอธิบาย, สิ่งที่ลูกค้าจะได้รับ (และต่อด้วยแก้ไข / หมายเหตุ)

แนวคิด: ราคาหลักอยู่นอกพับ — รายละเอียดยาวอยู่ใน accordion

## 3) สร้างใบเสนอราคา — 2 สเต็ป

```
[1 ระบุข้อมูล] ——— [2 ตรวจสอบ]
```

### สเต็ป 1 — ระบุข้อมูล (ซ้ายฟอร์ม / ขวาสรุป)

**ซ้าย**
- แบนเนอร์แบบรอบเดียว
- รูปแบบการจ้าง (บุคคล/บริษัท) — ถ้าบริษัทยังไม่ผ่าน KYC ให้มีกล่องแจ้งเตือน
- ระบุราคา (บาท)
- **ระยะเวลาการทำงาน** — วันเริ่ม–วันจบ (ของเรา: ช่องวันที่บาง ไม่ต้องมีพื้นหลัง/กรอบหนา)
- ข้อมูลเพิ่มเติม (ไม่บังคับ) — พับได้
- เลือกงาน / ผูกโปรเจกต์จากพอร์ต (มี thumbnail + เปลี่ยน)

**ขวา — สรุปเงิน (Fastwork มีค่าธรรมเนียมระบบ)**
- ราคางาน
- ค่าธรรมเนียมระบบ (−)
- **เงินที่คาดว่าจะได้รับ** (ไฮไลต์)

รอบแรกของเราอาจสรุปแค่ ยอดรวม / มัดจำ / หัก ณ ที่จ่าย — ค่าธรรมเนียมแพลตฟอร์มค่อยใส่ตอนมีเงินจริง

**Footer:** บันทึกเป็นเทมเพลต · ดำเนินการต่อ

### สเต็ป 2 — ตรวจสอบ

พรีวิวก่อนส่ง (ของเรามีแล้ว: กดส่ง → เช็คพรีวิว → ยืนยันส่ง)

## 4) กติกา product ที่ล็อกไว้แล้ว (Aplus1)

| หัวข้อ | ทิศทาง |
|--------|--------|
| จุดเข้า | ปุ่มเดียวในแชท hire — ไม่แยก «เอกสารเต็ม So1o» ในรอบนี้ |
| รายการ | บรรทัดๆ แบบบัญชี (ชื่อ / จำนวน / ราคาหน่วย / รวม) + เพิ่มรายการ |
| ลูกค้า | ชื่อ, อีเมล, โทร, ที่อยู่, เลขผู้เสียภาษี — ดึงจาก hire ได้ |
| มัดจำ | 50% · จ่ายเต็ม · กำหนดเอง % (แถวเดียว) |
| ไทม์ไลน์ | วันเริ่ม–จบ + งวดงาน; ช่องวันที่งวดไม่มีพื้นหลัง/กรอบ |
| พรีวิว | ไม่แยกคอลัมน์ตลอดเวลา — ปุ่มดูพรีวิว / ยืนยันก่อนส่ง |
| Launch | ปิดไว้ก่อนผ่าน flow ใช้เว็บรอบแรก + การเงิน |

## 5) สิ่งที่ยังไม่ทำในรอบนี้

- Escrow / ค่าธรรมเนียมระบบ / จ่ายจริง
- ใบแจ้งหนี้ + ใบเสร็จ (แผน Phase ถัดจาก offer)
- เทมเพลตบันทึกซ้ำใช้ได้หลายลูกค้า (UI Fastwork มี — ของเรายังไม่มี)
- รูปแบบจ้างบริษัท + KYC บริษัท
